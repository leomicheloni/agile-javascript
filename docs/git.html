Conceptos:

  • Trabajamos simpre sobre una copia local del repositorio remoto
  • El proceso de "subir" nuestros cambios se realiza en dos partes: commit y push
  • No podemos subir cambios si no tenemos la última versión del código localmente
  • En caso de bajar cambios y tener diferencias GIT hace un Merge automático y un commit a nuestro repo local
  • Si GIT no puede resolver los comflictos lo hacemos nosotros, en tal caso cuando querramos hacer commit veremos todos los archivos cambiados
  • Es muy simple hacer branches, de hecho es la práctica a seguir
  • Podemos bajar cualquier revisión fácilmente
  • Al cambiar de branch seguimos trabajando sobre el mismo lugar
