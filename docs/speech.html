Agilidad

Qué es la agilidad?
Qué problemas permite afrontar?
Cuáles son las metodologías más difundidas?


    

  • Scrum

Surge de la necesidad de llevar adelante proyectos donde el ambiente es cambiante.
Es más forma de gestionar la relación del equipo, las roles y el potencial de las personas.
La agilidad también es:

responde rápido a los cambios

Hay un problema

El problema es que para soportar todos esos principios hace falta apoyarse en técnicas (llamemoslás costumbres)

Por ejemplo

Es difícil soportar cambios continúos si nuestra arquitectura no escala
Tenemos que seguir principios de arquitectura

Es muy costoso tener que hacer ciertas tareas a mano
tenemos que automatizar el despliege del soft y otras tareas (delivery continuo)

A medida que crece la complejidad del proyecto es más importante hacer pruebas de regresión
tenemos que automatizar pruebas unitarias

Es costos probar todo de nuevo, y es complejo probar la UI de manera consistente
Prueba de UI automáticas o semi automáticas

Que no haya partes "oscuras" del código y tener propiedad colectiva del código.
